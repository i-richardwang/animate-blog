> 通过浏览器端 Python 运行时实现完全本地化的 Excel 分析工具。用户使用自然语言描述需求，AI 生成 Python 代码在本地执行，完成数据清洗、分析和可视化。从技术架构上保证敏感数据不离开用户设备。

处理财务报表、员工信息、客户数据时，将文件上传到云端总会产生隐私担忧。ExcelWhisper 将计算能力部署在浏览器端，数据始终保留在用户本地，服务器只接收表结构的元数据（字段名称和类型），无法获取具体的数据值。

![ExcelWhisper 主界面](/projects/excelwhisper/main-interface.png)

上传 Excel 或 CSV 文件后，通过自然语言描述分析需求。例如"按部门统计员工人数"、"筛选销售额大于 5000 的订单"、"生成月度销售趋势图"。系统会生成对应的 Python 代码，在浏览器中执行并返回结果。

## 本地化的数据处理架构

基于 Pyodide 在浏览器中运行完整的 Python 环境。上传的文件内容存储在浏览器内存中，所有 pandas 计算和 Plotly 图表生成都在本地完成。

服务器接收的只是表结构的抽象描述，例如「员工ID: 文本，入职日期: 日期，薪资: 数字」。某个员工的具体 ID 是什么、薪资金额是多少，服务器完全无从得知。产品提供"隐私检查器"功能，用户可以随时查看服务器实际接收的信息内容。

技术架构从根本上杜绝了数据泄露的可能。

![数据处理结果展示](/projects/excelwhisper/result-table.png)

处理完成的结果以表格形式呈现，支持分页浏览和排序。用户可以将结果下载为 Excel 或 CSV 格式，继续在本地使用。

## AI 生成代码而非直接接触数据

工作流程是：用户的自然语言需求和上传文件的表结构发送到服务器，AI 根据表结构生成 Python 代码并返回浏览器，代码在本地执行，结果呈现给用户。

AI 只需要知道表中有哪些字段、字段的数据类型，就能生成正确的 pandas 和 Plotly 代码。无需接触实际的数据值。利用大模型的代码生成能力，隐私保护和功能实现找到了平衡点。

## 多语言支持和可视化主题

系统自动识别用户输入的语言（中文、英文、日语等），整个交互过程使用相同的语言响应。生成的图表支持多种配色主题，用户可以根据使用场景灵活切换。

![饼图可视化（默认主题）](/projects/excelwhisper/pie-chart-default.png)

![饼图可视化（海洋主题）](/projects/excelwhisper/pie-chart-ocean.png)

数据可视化基于 Plotly 生成，支持柱状图、折线图、饼图、散点图等常见图表类型。图表尺寸可以调整，可以下载为图片文件。

## 智能的需求理解机制

系统在生成代码前会进行需求预判，避免无效执行。如果用户只是说"分析一下数据"，系统会主动询问："请告诉我您想分析哪个字段"。如果需求涉及不存在的字段，会提前提醒而不是执行后报错。如果请求超出数据处理范围，会明确说明系统边界。

预判机制减少用户的试错成本，明确系统的能力边界。

## 智能的文件输出管理

处理完成需要保存时，系统根据操作内容自动生成有意义的文件名。筛选后的销售数据保存为 `filtered_sales.csv`，月度统计保存为 `monthly_stats.csv`。只保存最终结果，不产生中间临时文件，减少用户的文件管理负担。

## 自然语言替代函数语法

用户无需记住 VLOOKUP、SUMIFS 这些复杂的函数语法。直接说"在客户表中找到对应的客户，从订单表获取他们的销售额"，系统会生成相应的 JOIN 操作代码并执行。

支持多文件联合分析、数据清洗、图表生成、统计计算等常见数据处理任务。产品定位在自动化重复性的 Excel 操作，深度分析和业务判断由用户完成，AI 负责处理繁琐的技术细节。
