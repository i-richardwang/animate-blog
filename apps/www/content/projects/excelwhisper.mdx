> 隐私优先的 Excel 分析工具，所有数据处理都在浏览器本地完成。用自然语言描述需求，AI 生成 Python 代码在本地执行，完成数据清洗、分析和可视化。从技术架构上保证敏感数据不离开设备。

处理财务报表、员工信息、客户数据时，上传到云端总让人不安。传统的 Excel 分析工具要么需要安装插件，要么需要把文件上传到服务器。

ExcelWhisper 通过自然语言对话完成数据处理任务，所有计算都在浏览器本地执行，从技术架构上保证数据不离开用户设备。

## 浏览器内的 Python 运行时

使用 Pyodide 在浏览器里运行完整的 Python 环境。用户上传的 Excel 文件内容存储在浏览器内存中，所有数据处理（pandas 计算、图表生成）都在本地执行。

服务器只接收文件的元数据：字段名称和数据类型，比如「客户ID: 整数，销售额: 浮点数」。不会看到具体的客户 ID 是多少，销售额是多少。产品提供"隐私检查器"功能，用户可以随时查看服务器实际接收了哪些信息。

这不是承诺，而是技术上做不到获取数据。

## AI 生成代码而非分析数据

工作流程是：用户输入需求（"找出销售额大于 1000 的客户并生成柱状图"）→ 发送需求和表结构到服务器 → AI 生成 Python 代码 → 返回代码到浏览器 → Pyodide 本地执行代码 → 显示结果。

AI 只需要知道表里有哪些字段、什么类型，就能生成正确的 pandas 和 plotly 代码。不需要看到实际数据值。生成的代码在用户浏览器执行，计算结果直接在本地显示。

这种设计的巧妙之处在于利用了大模型的代码生成能力，而不是让它直接分析数据。隐私和功能得到了平衡。

## 自然语言替代 Excel 函数

不需要记住 VLOOKUP、SUMIFS 这些函数的语法。直接说"在表1中找到客户，从表2获取他们的销售额"，系统会生成对应的 JOIN 操作代码并执行。

支持多文件联合分析、数据清洗、图表生成等常见任务。产品定位很明确：自动化重复性的 Excel 操作，而不是提供业务洞察。深度分析和判断留给用户，AI 只负责处理繁琐的技术细节。
