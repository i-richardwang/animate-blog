由大语言模型驱动的 AI 研究工具，帮助用户进行深度主题研究和报告生成。LiteResearch 通过智能 Agent 和并行信息检索，自动化从查询分解到结构化报告生成的整个研究过程。

## 项目背景

基于 [gpt-researcher](https://github.com/assafelovic/gpt-researcher) 开发的简化版本。保留核心概念——通过 AI Agent 实现智能信息检索和报告生成，但简化了实现方式，去除复杂功能模块，使代码结构更清晰易懂。

## 系统架构

![系统架构](/projects/literesearch/architecture.png)

LiteResearch 采用智能研究工作流设计，从用户输入研究主题开始，自动选择专业 AI Agent 角色（如金融分析师、技术专家），生成多个相关子查询以确保研究全面性。

系统并行处理所有子查询，每个子查询独立进行网络搜索、内容提取和压缩。使用向量化技术筛选最相关的内容，避免 LLM 上下文限制。根据报告类型决策，生成结构化的研究报告，包含引言、分主题分析、结论和参考文献。整个流程通过 Langfuse 监控，可追踪每个阶段的处理结果。

## 使用流程

### 1. 研究设置

![研究设置界面](/projects/literesearch/research-settings.png)

在研究设置界面中输入主题（如"AI in Human Resources"），选择报告类型（分析性、综合性、资源摘要等）和语气风格（正式、非正式、分析性等）。

高级设置允许调整最大子查询数（1-10）、最大子主题数（1-18）和每个查询的最大结果数（1-20），根据研究需求控制深度和广度。点击"Start Research"后，系统显示实时进度，包括 Agent 选择、子查询生成、并行搜索等步骤，全程自动化。

### 2. 报告生成与展示

![研究报告展示](/projects/literesearch/report-result.png)

研究完成后生成完整的结构化报告。报告包含 Executive Summary（高层次概述）、分主题分析章节（如 AI in Recruitment、Functional Benefits、Critical Evaluation）、详细内容（当前趋势、功能优势、关键数据）、引用来源链接和完整参考文献。

报告支持下载为 Markdown 格式，内容结构清晰、逻辑连贯，适合直接用于学术研究、市场分析或技术报告。

## 核心特性

**并行处理架构**  
同时处理多个子查询，通过并行网络搜索和内容提取显著提升研究效率，相比串行处理大幅缩短完成时间。

**向量化内容压缩**  
使用 Embeddings 技术智能筛选最相关的内容，解决 LLM 上下文窗口限制问题，确保关键信息不丢失。

**灵活的报告定制**  
支持 6 种报告类型（综合研究、资源摘要、研究大纲、详细深度、自定义、子主题）和 6 种语气风格（正式、非正式、分析性、说服性、信息性、解释性），满足不同场景需求。

**完整的可观测性**  
集成 Langfuse 监控全链路 LLM 调用，追踪每个阶段的性能指标和成本，便于优化和调试。

**清晰的代码结构**  
基于 gpt-researcher 简化实现，去除复杂模块，保留核心逻辑。模块化设计使代码易于理解、维护和扩展。

## 技术栈与架构

### 技术组件

- **Frontend**: Streamlit 提供简洁的 Web 界面
- **Backend**: Python + LangChain 实现 AI 研究工作流
- **Search Engine**: Tavily API 提供网络搜索能力
- **Vectorization**: Embeddings 实现内容压缩和相关性筛选
- **Monitoring**: Langfuse（可选）监控 LLM 调用和性能

### 项目结构

```
literesearch/
├── frontend/                   # 前端应用
│   ├── literesearch_app.py    # 主应用（核心功能逻辑）
│   ├── ui_components.py       # UI 组件和样式定义
│   └── assets/                # 静态资源文件
├── backend/                   # 后端核心
│   └── literesearch/          # 文献研究模块
├── utils/                     # 工具函数
│   ├── llm_tools.py          # LLM 工具类
│   └── langfuse_tools.py     # Langfuse 监控工具
└── pyproject.toml            # 项目配置和依赖
```

### 模块化设计

**职责分离**: `literesearch_app.py` 处理核心功能逻辑，包括用户交互和 AI 研究工作流；`ui_components.py` 负责 UI 组件和样式，处理界面显示和用户体验。这种清晰的职责分离使代码易于维护和扩展。

**核心功能模块**: 包括研究设置和参数配置、AI 研究工作流控制、报告生成和下载、异常处理和用户反馈、LLM 调用监控和分析。
