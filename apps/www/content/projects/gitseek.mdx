> 从任意 GitHub 仓库中快速提取完整的相关代码。用自然语言描述需求，AI 自动定位文件并格式化成适合 Claude、Cursor 等工具的输入格式。支持项目结构可视化和私有仓库访问，帮助快速理解和复用开源项目。

看开源项目学习时，找特定功能的实现总要在文件间跳来跳去。想知道用户认证怎么做的，需要翻遍 controllers、middleware、utils 好几个目录。想把代码喂给 Claude 分析，又得手动整理成合适的格式。

GitSeek 用自然语言描述需求，AI 自动定位相关文件，生成适合 AI 工具阅读的格式。

## AI 理解项目结构

描述"查找用户认证相关的代码"，系统会分析整个仓库，找出登录、注册、权限验证等相关的完整文件。不只是搜索关键词，而是理解代码的语义和调用关系，确保不遗漏关键部分。

支持各种具体需求：支付流程与 Stripe 集成、React 状态管理与 Context 配置、UI 组件的布局和动画、错误处理机制、性能优化方案等。

## 一键复制给 AI 工具

提取的代码会自动格式化成 Claude、Cursor 等工具的最佳输入格式。包含文件路径、代码内容、项目目录树。直接复制粘贴就能让 AI 理解项目结构，进行解读或改造。

同时生成可视化的架构图，帮助快速掌握项目整体结构。

## 安全处理私有仓库

支持私有仓库访问。GitHub Token 在分析过程中全程加密，使用完立即删除，不会存储。下载的仓库文件在处理后也会及时销毁。系统只记录使用次数，不保存代码内容。

## 配置自己的大模型 API

免费版每天可以进行 5 次提取。如果配置自己的大模型 API（OpenAI、Claude、或任何兼容 OpenAI 格式的服务），可以解锁无限使用次数。用户完全掌控 AI 使用成本，同时享受完整功能。
