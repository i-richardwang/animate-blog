> 专为 AI 编程时代设计的智能代码提取工具。用自然语言描述需求，AI 自动定位完整的相关代码并格式化成适合 Claude、Cursor 等工具的输入格式。支持项目架构可视化和私有仓库访问，让代码学习和复用变得前所未有的高效。

学习开源项目时，找特定功能的实现往往需要在几十个文件间跳来跳去。想了解用户认证是如何实现的，需要翻遍 controllers、middleware、utils 好几个目录，还要确保不遗漏关键的依赖文件。更麻烦的是，即使找到了代码，要将其整理成适合 AI 工具分析的格式仍然是繁琐的手动工作。

GitSeek 将这个过程简化为一句话：描述你想要什么，AI 自动帮你找到并整理好。

## 系统架构

![系统架构流程图](/projects/gitseek/GitSeek.png)

GitSeek 采用智能的图工作流架构，从用户输入 GitHub 仓库 URL 开始，通过自然语言描述提取需求。系统首先克隆仓库并扫描目录结构，然后根据用户需求进行智能决策。

**智能过滤决策节点** - 当用户提供自然语言查询时，AI 自动生成文件过滤规则；当用户提供明确的过滤规则时，直接使用用户规则；如果不需要过滤，则提取所有代码内容。

**架构图生成决策节点** - 如果用户勾选生成架构图选项，系统会并行分析项目结构并生成可视化的 Mermaid 架构图；否则跳过此步骤，直接进入代码提取流程。

**提取代码内容** - 应用过滤规则筛选相关文件，读取内容并格式化成 AI 友好的输出格式，包含文件路径标识、代码内容和项目目录树。

**清理资源** - 处理完成后立即删除克隆的临时仓库文件，确保用户隐私和服务器存储空间的整洁。

整个流程通过智能路由控制，灵活支持不同的使用场景，同时保证处理效率和输出质量。

## 功能演示

### 1. 快速开始

![GitSeek 主界面](/projects/gitseek/main-interface.png)

在主界面输入 GitHub 仓库 URL，可以是任何公开仓库或私有仓库（需提供 Token）。系统支持多种输入方式：

- **自然语言查询**：直接描述需求，如"了解项目的 landing page 设计，但不需要 shadcn 等组件"
- **明确过滤规则**：可以留空让系统提取整个仓库
- **架构可视化**：勾选实验性功能，生成项目架构图（使用 GPT-4 等高级模型效果更好）

点击"Extract Code Now"按钮，系统开始智能分析和提取代码。

### 2. 提取策略展示

![提取策略和结果概览](/projects/gitseek/extraction-results.png)

提取完成后，系统展示详细的处理结果。左侧显示提取策略说明：

- **File Matching**：采用的匹配方式（include 包含模式或 exclude 排除模式）
- **Matching Patterns**：具体的文件匹配规则列表
- **策略解释**：AI 说明为什么选择这些规则

右侧显示概览信息：

- **Repository**：仓库标识符
- **Matched Files**：成功匹配的文件数量
- **Estimated Tokens**：预估的 Token 消耗量，帮助控制 AI API 成本

支持一键下载完整结果，方便本地保存和使用。

### 3. 架构可视化

![系统设计架构图](/projects/gitseek/architecture-diagram.png)

当启用架构可视化功能时，AI 会自动分析项目结构并生成清晰的系统设计图。图表展示：

- **Frontend 层**：用户界面、认证、支付、核心功能等前端组件
- **Backend 层**：API Gateway 和各类服务（Auth Service、Payment Service 等）
- **External Services 层**：第三方集成（Clerk、Stripe、OpenAI、AWS S3 等）
- **Database 层**：数据存储方案（PostgreSQL 等）

不同层次使用不同颜色区分，连接线标注交互关系，帮助快速建立项目全局认知。架构图支持全屏查看和缩放操作。

### 4. 代码内容输出

![仓库结构和代码内容](/projects/gitseek/code-output.png)

最下方展示完整的提取结果，分为两部分：

**Repository Structure** - 清晰的目录树展示项目文件组织结构，包含所有文件和目录的层次关系。一键复制按钮方便快速使用。

**Files Content** - 格式化的代码内容，每个文件都带有清晰的路径标识和分隔符。输出格式专为 AI 工具优化，可以直接复制粘贴到 Claude、ChatGPT 或 Cursor 中进行深度分析、代码解读或功能改造。

## 核心特性

**自然语言代码发现**  
不需要了解项目结构，用日常语言描述需求即可精准定位代码。系统会自动理解"用户认证"、"支付流程"、"状态管理"等模糊表述，智能生成最优的文件匹配策略。支持正向包含和反向排除两种模式，AI 自动选择需要最少规则的方案。

**智能过滤策略**  
系统不是简单的关键词搜索，而是理解代码语义和项目结构。当目标明确时使用精准匹配，当表述模糊时倾向宽泛匹配确保不遗漏。自动排除 node_modules、.git、测试文件等干扰内容，同时保留所有业务相关代码。

**架构可视化**  
自动分析项目类型、组件关系和技术栈，生成 Mermaid 格式的系统架构图。垂直布局展示用户→前端→后端→数据库的层次结构，不同形状和颜色区分组件类型，清晰标注模块间的调用关系。帮助新成员快速理解项目架构，适合技术评审和文档生成。

**AI 工具无缝对接**  
输出格式专为 Claude、ChatGPT、Cursor 等 AI 工具优化。包含文件路径、完整代码、项目目录树和架构图，提供充足的上下文信息。预估 Token 消耗量，避免超出模型限制或产生意外成本。一键复制或下载，直接喂给 AI 工具进行代码分析、功能实现或重构建议。

**私有仓库安全访问**  
支持企业和个人私有仓库。GitHub Token 在传输和使用过程中全程加密，只用于单次 Git 克隆操作。仓库文件仅在内存中处理，任务完成后立即删除临时文件。系统不存储任何代码内容，只记录使用次数等非敏感元数据，从技术架构上保障代码安全。

## 适用场景

适合学习开源项目时快速定位特定功能的实现，理解陌生代码库的整体架构。也可用于复用功能模块到自己的项目中，或进行代码安全审计和技术方案评审。

通过自然语言交互和智能过滤，将代码查找和整理的时间从半小时缩短到几分钟。
