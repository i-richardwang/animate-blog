基于大语言模型和向量语义搜索的智能人才推荐系统，实现从自然语言招聘需求到精准候选人匹配的完整流程。TalentMatch 通过对话式需求澄清、智能搜索策略生成和多维度评分机制，突破传统关键词匹配的局限，深入理解候选人的真实能力和经验。

## 系统架构

![系统架构图](/projects/talentmatch/architecture.png)

TalentMatch 采用四阶段智能处理流程，从用户输入招聘需求开始，通过 LLM 驱动的对话式交互逐步澄清和细化需求。系统随后生成多维度的搜索策略，为不同简历字段（工作经历、技能、教育背景等）分配动态权重。

**需求澄清模块** - 通过 LLM 分析初始需求的完整性，主动提问补充关键信息（职位、技能、经验要求等），迭代对话直到获得清晰完整的招聘画像。

**策略生成模块** - 两阶段策略制定：首先确定各简历维度的整体重要性权重，然后针对每个维度生成具体的向量查询文本。根据职位特点动态调整搜索重点，实现定制化匹配。

**向量检索引擎** - 基于 Milvus 向量数据库的语义搜索系统。将简历各部分（工作描述、技能、项目经验等）预先向量化存储，通过语义相似度而非关键词匹配找到真正相关的候选人。

**评分与推荐** - 多维度加权评分机制，整合各字段的语义匹配度。LLM 生成每位候选人的推荐理由，解释匹配的具体原因，提升推荐透明度和可信度。

## 使用流程

### 1. 对话式需求澄清

![需求澄清对话](/projects/talentmatch/requirement-clarification.png)

输入初始招聘需求后，系统通过 LLM 分析需求的完整性。如果信息不足（如缺少经验要求、具体职责等），系统会主动提出针对性问题引导补充。每轮对话后实时更新需求理解，避免反复提问。

用户可以用自然语言描述需求，无需填写固定字段。系统会自动提取关键信息维度：职位类型、核心技能、工作年限、行业背景、教育要求等。对话过程简洁高效，通常 2-3 轮即可完成需求确认。

### 2. 智能策略生成

![搜索策略生成流程](/projects/talentmatch/search-strategy.png)

需求确认后，系统通过两阶段策略生成机制制定个性化搜索方案。第一阶段确定各简历维度的整体重要性权重（如工作经历 40%、专业技能 35%、项目经验 20%）。第二阶段针对每个维度生成具体的向量查询文本和内部权重。

图中展示了完整的策略生成流程：顶部显示维度权重分配，中间展示各维度的具体查询向量（包含查询内容和权重），底部汇总为最终的系统检索执行方案。这种两层权重体系使系统能根据不同职位要求动态调整搜索重点，实现真正的定制化匹配。

### 3. 候选人推荐与详情

![推荐结果展示](/projects/talentmatch/recommendation-results.png)

搜索完成后展示 Top N 候选人列表，每位候选人配有：

- **综合匹配分数**：基于多维度语义匹配的量化评分
- **推荐理由**：LLM 生成的简洁说明，解释匹配的关键点（相关经验、技能契合度、教育背景等）
- **核心信息**：工作年限、技能概览、教育背景等关键指标
- **详细简历**：展开查看完整的工作经历、项目经验和技能详情

推荐理由突出候选人与职位要求的具体匹配点，帮助招聘者快速理解推荐依据，提高筛选效率。

## 核心特性

**对话式需求理解**  
通过与 LLM 的自然对话逐步细化招聘需求，相比传统的表单填写更灵活自然。系统能够捕捉隐含需求和细微偏好，生成精准的招聘画像。主动澄清机制确保信息完整性，避免模糊需求导致的匹配偏差。

**定制化搜索策略**  
根据职位特点动态生成搜索策略，为不同简历维度分配差异化权重。技术岗位侧重技能和项目经验，管理岗位关注领导力和团队管理。两层权重体系（维度间权重和维度内权重）实现千人千面的定制化检索。

**语义匹配能力**  
使用向量 Embedding 技术理解简历和需求的深层语义，突破关键词匹配的局限。即使表述方式不同，相似的技能和经验也能被准确识别。支持同义词理解和概念推理，减少因表达差异导致的人才遗漏。

**多维度评分**  
综合评估候选人在工作经历、技能、教育背景、项目经验等多个维度的匹配度。评分机制整合向量相似度和策略权重，准确反映简历与特定职位的契合程度。避免单一维度评估的片面性。

**可解释推荐**  
为每位推荐候选人生成清晰的推荐理由，说明匹配的具体原因。提升系统透明度和决策可信度，帮助招聘者理解推荐逻辑，快速把握候选人亮点。支持招聘者优化需求描述，形成反馈闭环。

## 技术架构

### 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| **前端** | Streamlit | Web 交互界面 |
| **AI 框架** | LangChain | LLM 调用和工作流管理 |
| **向量数据库** | Milvus / Zilliz Cloud | 语义搜索和相似度匹配 |
| **关系数据库** | MySQL | 简历数据存储 |
| **Embedding** | Text Embedding API | 文本向量化 |

```
TalentMatch/

│       ├── recommendation/        # AI 推荐系统
│       │   ├── clarification.py   # 需求澄清
│       │   ├── strategy.py        # 策略生成
│       │   ├── scoring.py         # 评分机制
│       │   └── explanation.py     # 推荐理由生成
│       └── storage/               # 数据存储
├── utils/                         # 工具模块
│   ├── ai/                        # LLM 和 Embedding 客户端

└── scripts/                       # 数据导入和初始化
```

## 适用场景

招聘经理快速筛选海量简历、HR 团队提升人才匹配精准度、猎头顾问发现潜在候选人、企业内部人才搜索和调配、招聘流程自动化和效率提升。

系统特别适合需要深度理解能力和经验的职位匹配场景，相比传统关键词筛选能够显著减少优质人才的遗漏，提高推荐质量和招聘效率。
