当我决定搭建个人网站时，不想只是用个基础的博客模板了事。我需要一个既能作为笔记的数字花园，又能展示作品的空间，同时还要好看、现代。

后来发现了 [Animate UI](https://animate-ui.com)。

## 为什么选择 Animate UI？

Animate UI 是一个用 Tailwind CSS 和 Motion 构建的开源 React 组件库。吸引我的不只是那些精美的动画效果，而是它背后的理念。

和普通的 npm 包不同，Animate UI 采用的是 shadcn/ui 推广的"复制并定制"方式。你不是把它安装成依赖项，而是直接把源代码复制到项目里。这意味着你对每个组件、每个动画、每个设计决策都有**完全的控制权**。

Animate UI 原本是作为组件库的文档站点设计的，用来展示动画原语、组件和图标。但我看到了不同的可能性。

## 改造过程

### 从组件库到数字花园

Animate UI 架构的优势在于灵活性。站点的技术栈包括：

- **Next.js 15** 作为框架
- **Fumadocs** 提供文档结构
- **Tailwind CSS** 负责样式
- **Motion**（Framer Motion）实现动画

这个技术栈很适合做个人站点。文档结构可以变成知识库，组件展示可以变成博客，那些精美的动画能让整个体验更愉悦。

### 核心改动

改造的思路很简单：**保留原有结构，定制内容**。

**导航和结构**：原站点有 Components、Primitives、Icons 这些板块。我把它们换成了自己的内容支柱：AI 探索、数据科学、开发。同时还加了一个专门的 Blog 板块，这是原项目没有的。好在 Fumadocs 本身就支持博客集合，只需要接入并添加到导航里。

**首页演变**：Hero 区域的动画 logo 和文字效果太棒了，舍不得丢。我保留了所有动画，但把文案从"Animate your UI"改成了"Learning, Building, Sharing"，更适合个人站点。CTA 按钮也从指向组件文档改成了"探索笔记"和"阅读博客"。

**内容架构**：用一个双轨内容系统替代了组件文档：

- **知识库**（`/docs`）：三层结构，包括主模块（AI、数据科学、开发）、子分类和详细笔记
- **博客**（`/blog`）：用于分享想法、教程和项目更新，比知识库更具叙事性、结构也更松散

原有的 meta.json 组织系统对结构化笔记和按时间排序的博客文章都很好用。

### 设计决策

有一点我很谨慎：**保持原有的设计语言**。动画、间距、排版都是精心设计的。与其重新造轮子，不如让内容适应现有的美学风格。

结果就是一个连贯、专业且动画独特的个人站点。

## 定制你自己的站点

如果你想用这个作为起点搭建自己的站点，需要定制的内容主要是配置和内容更新，相对直观。

### 站点身份

第一步是让它成为你的站点，包括品牌、元数据和链接。

**更新站点元数据**，在 `apps/www/app/layout.tsx` 中：

```tsx
export const metadata: Metadata = {
  title: {
    template: '%s - Your Name',
    default: 'Your Name - Your Tagline',
  },
  description: 'Your site description...',
  // Update keywords, authors, publisher, etc.
};
```

**替换域名**，在 `layout.tsx` 和 `lib/json-ld.ts` 中：

```tsx
// 查找并替换
url: 'https://richardwang.me';
// 改为
url: 'https://yourdomain.com';
```

**更新社交链接**，GitHub 和 Twitter/X 链接出现在多个文件中：

- `components/header.tsx`
- `components/docs/nav.tsx`
- `components/footer.tsx`
- `app/docs/layout.tsx`

搜索 `i-richardwang` 和 `richard2wang`，替换成你的用户名。

### 视觉标识

**Logo**：如果你想保留动画 SVG logo（推荐，效果很棒），需要把你的文字创建成 SVG 路径。

最简单的方法：

1. 打开 Figma，创建文字
2. 选择字体和大小
3. 右键 → "Outline Stroke"
4. 导出为 SVG
5. 把路径数据复制到 `components/logo.tsx`

Logo 组件用 Motion 来动画化路径绘制，效果很流畅。

**Favicon**：替换 `/public` 中的文件：

- `favicon-16x16.png`
- `favicon-32x32.png`
- `apple-touch-icon.png`
- `og-image.png`（用于社交分享）

### 内容和导航

**首页 hero**（`components/hero.tsx`）：

```tsx
const TITLE = 'Your Tagline Here';

// 更新徽章/公告
<Link href="/blog">Latest thoughts and notes</Link>

// 更新描述
<p>Your personal introduction...</p>

// 更新 CTA 按钮
<Link href="/docs">Explore Notes</Link>
<Link href="/blog">Read Blog</Link>
```

**顶部导航**（`components/docs/nav.tsx`）：

```tsx
export const NAV_ITEMS = [
  { title: 'Your Section 1', url: '/docs/section-1' },
  { title: 'Your Section 2', url: '/docs/section-2' },
  { title: 'Blog', url: '/blog' },
];
```

**侧边栏标签**（`lib/sidebar.tsx`）：

```tsx
export const SIDEBAR_TABS = [
  {
    title: 'Your Section',
    description: <TabsDescription title="Description" count={0} />,
    icon: <YourIcon />,
    url: '/docs/your-section',
  },
  // 为知识库添加更多板块
];
```

**侧边栏指南链接**（`app/layout.config.tsx`）：

```tsx
links: [
  {
    type: 'separator',
    name: 'Guide',
    icon: <BookOpen />,
  },
  {
    text: 'Your Link',
    url: '/your-url',
  },
];
```

### 内容组织

站点支持两种类型的内容，各有各的用途：

**知识库**（`/content/docs/`）：按主题组织的结构化、层级化笔记。每个板块需要：

- `index.mdx` - 着陆页，用 Cards 链接到子板块
- `meta.json` - 定义导航结构和排序
- 单独的 `.mdx` 文件存放详细笔记

`meta.json` 示例：

```json
{
  "title": "Your Section",
  "pages": ["---Category Name---", "category/article-1", "category/article-2"]
}
```

**博客**（`/content/blogs/`）：按时间排序的文章，用于分享想法、教程和更新。每篇文章是一个独立的 `.mdx` 文件，带有 frontmatter：

```mdx
---
title: Your Post Title
description: Brief description
date: 2024-01-15
tags: ['tag1', 'tag2']
---
```

双轨结构提供了灵活性：用 `/docs` 存放常青的、结构化的内容，用 `/blog` 存放时效性强的、叙事性的文章。

### 关于代码保留

在代码库中，你会看到这样的注释：

```tsx
// Original ... (commented out for upstream sync)
```

我保留了所有原始 Animate UI 代码的注释。这样可以：

- 看到做了哪些改动
- 同步上游更新
- 参考原始实现

你可以保留这个模式，也可以在熟悉代码库后删除注释。

## 改造结果

从组件库变成了个人数字空间。曾经用来展示 UI 组件的动画，现在引导访客浏览我的想法和项目。曾经用来解释原语的文档结构，现在组织我的知识。

最棒的是什么？因为所有东西都可以在源码层面定制，我可以持续演进它。想加新板块？创建一个新模块。需要不同的布局？修改组件。可能性是无限的。

如果你用它搭建了什么，我很想看看。欢迎联系我分享你的创作！
